Модуль копируемый из проекта в проект. Зачастую мне лень создавать отдельный модуль на проекте и я докидываю в этот модуль что-то нужное на одном проекте. А на следующем подчищаю. Не надо так.

Что у нас тут есть.
## JS
Я скопировал его в корень, и немного подчистил. Там же есть readme.

## CSS
Минимальный css для аякс форм. Чтобы текст ошибок, сообщений появлялся и исчезал когда надо. Хотя предполагается что верстальщик сделает все это куда более красивым.

## lib

В корне просто классы. В основном это orm таблицы. Конечно скидывать случайные таблицы смысла не несет. Так что оставил только фигню для списков товаров для юзеров. Это такая две таблицы, которые наследуются от `useritemslist`. Также таблицу ~~расширяющую~~ привязанную к разделу. Просто ради идеи. Если вам чего-то не хватает в таблицах битрикса, то просто сделайте свою таблицу, привязанную как 1:1. А еще есть набор утилит. Просто нужные функции. Всякие обертки вокруг отложенных функций, и прочее.

### controller
Тут методы, которые можно вызвать через ajax. В данном случае работа со списками избранного, корзины. Правда часть отключена, да и шаблон корзины отдающий arResult как json еще надо создать, что впрочем не сложно.

Зато можно подсмотреть как создавать подобные штуки. Подсказка, нужен еще файлик .settings.php по образцу.

### events
Название говорит само за себя.

Содержит пару вечно нужных вещей. Это пдфка счета и модификация списка товаров для нового заказа.

### fixes
Как-то мне не очень понравился один избыточный валидатор из коробки от битры. И я решил сделать свой orm класс, не такой строгий.


## wk
Бинарник для генерации pdf из html. За документацией, обновлениями и прочим гуглить wkHtmlToPdf.


## options.php
В настройках модуля можно делать кнопки. Например переустанавливающие таблицы модуля. Вообще там было больше, но я подчистил.
